# Ink 7 Migration

**Date:** 2026-04-12
**Branch:** feature/ink-7
**Git strategy:** One branch, one commit per phase

## Context

The CLI dashboard (`packages/cli`) runs on Ink 6.8.0. Ink 7.0.0 was released on April 8, 2026 with new hooks, render options, and breaking changes. Our runtime already satisfies Ink 7's floor requirements (Node >=22, React 19.2.4), so the migration is feasible.

**Source:** https://github.com/vadimdemedes/ink/releases/tag/v7.0.0

## Goals

- Upgrade to Ink 7 with zero regressions
- Replace manual ANSI escape codes with Ink's `alternateScreen` option everywhere Ink safely covers the lifecycle; keep explicit handling only for tmux handoff if validation proves it necessary
- Add reactive terminal resize via `useWindowSize`
- Fail fast on compatibility issues before investing in refactors

## Non-Goals (deferred to follow-up)

- `usePaste` integration (spawn-dialog, auth modal, env manager)
- `useFocusManager` with `activeId` (replace manual focus state)
- Layout props (`maxWidth`/`maxHeight`, `useAnimation`, responsive improvements)

These are valuable but not required for the core Ink 7 upgrade. Each will be its own commit when we revisit.

## Phase 1: Compatibility Spike

**Scope:** Bump `ink` from `6.8.0` to `7.x` in `packages/cli/package.json`. Verify companion packages still work.

**Files:**
- `packages/cli/package.json` (version bump)
- Lockfile (regenerated by `bun install`)

**Steps:**
1. Update `ink` version in `packages/cli/package.json`
2. Run `bun install`
3. Run `bun run typecheck` from repo root
4. Run `bun run test` from repo root
5. Run `bun run lint` from repo root
6. Verify `@inkjs/ui` v2.0.0 imports work (check peer dependency ranges)
7. Verify `ink-link` v5.0.0 imports work

**Validation gate:** All tests pass, typecheck clean, lint clean. Zero behavior changes.

**Stop rule:** If `@inkjs/ui` or `ink-link` break under Ink 7, stop here and document the exact incompatibility before proceeding.

**Free wins from this phase:**
- Wide-character rendering fixes (CJK, emoji, overlapping characters) land automatically
- Kitty keyboard protocol queries all terminals instead of hardcoded allowlist

## Phase 2: Terminal Lifecycle Migration

**Scope:** Replace manual `\x1b[?1049h` / `\x1b[?1049l` ANSI escape codes with Ink's `{ alternateScreen: true }` render option. This phase covers both the dashboard bootstrap and the tmux attach/detach flow as a single risk surface.

**Files:**
- `packages/cli/src/commands/dashboard/usecase.ts` (~line 136) — dashboard bootstrap, raw ANSI entry/exit
- `packages/cli/src/tui/app.tsx` (~line 255) — attach/detach flow, mid-session alt-screen toggle
- `packages/cli/src/commands/dashboard/onboarding.tsx` — if it touches alt-screen

**Steps:**
1. Add `{ alternateScreen: true }` to the `render()` call in `usecase.ts`
2. Remove manual `\x1b[?1049h` / `\x1b[?1049l` writes from bootstrap path
3. Test tmux attach/detach flow under Ink's `alternateScreen`:
   - If Ink handles it cleanly: remove remaining ANSI writes from `app.tsx`
   - If not: keep manual ANSI only in the attach/detach path, document why
4. Test Ctrl+C / crash recovery — terminal must restore to normal screen

**Validation gate:**
- Dashboard launches in alternate screen, exits cleanly to normal terminal
- Tmux attach/detach works without terminal state leaks
- Ctrl+C from dashboard restores terminal correctly
- No raw ANSI codes remain (except possibly in attach/detach if Ink can't cover it)

## Phase 3: Reactive Sizing

**Scope:** Replace manual `useStdout().stdout` dimension reads with Ink 7's `useWindowSize` hook. The dashboard layout will re-render reactively when the terminal is resized, which currently doesn't happen.

**Primary file:**
- `packages/cli/src/tui/app.tsx` (~line 51) — the only file that reads terminal dimensions directly via `useStdout()`

**Downstream files (only if resize smoke test reveals they need changes):**
- `packages/cli/src/tui/dashboard.tsx` — consumes props/fixed layout constants
- `packages/cli/src/tui/status-bar.tsx` — consumes props/fixed layout constants
- `packages/cli/src/tui/MainPane/MainPane.tsx` — consumes props
- `packages/cli/src/tui/Sidebar/Sidebar.tsx` — consumes props

**Steps:**
1. Replace `useStdout()` dimension reads with `useWindowSize()` in `app.tsx`
2. Ensure layout computation uses the reactive values
3. Smoke test terminal resize — if downstream files need adjustment, update them
4. Test resize behavior at various terminal sizes

**Validation gate:**
- Resize terminal while dashboard is open — layout updates live
- No regressions at normal terminal sizes
- Sidebar and main pane proportions remain correct
- Status bar renders correctly after resize

## Breaking Changes Audit

| Change | Risk | Our Status |
|--------|------|------------|
| Node.js 22+ required | None | Already on `>=22` |
| React 19.2+ required | None | Already on `19.2.4` |
| Backspace sets `key.backspace` not `key.delete` | Low | `spawn-dialog.tsx` already checks `key.backspace \|\| key.delete` |
| Plain Escape no longer sets `key.meta` | Low | No code relies on `key.meta` for escape handling |
| `@inkjs/ui` compatibility | Unknown | Verify in Phase 1 |
| `ink-link` compatibility | Unknown | Verify in Phase 1 |
